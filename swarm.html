Docker Swarm: Scaling and Rolling Updates

Docker Swarm mode allows you to cluster multiple Docker hosts and manage them as a single virtual system. In this tutorial, you'll learn how to deploy a stack, scale services, and perform rolling updates.

Initializing Docker Swarm

First, initialize Docker Swarm on your manager node:


docker swarm init

Sample docker-compose.yml for Swarm

Create a file named docker-compose.yml with the following content:


version: "3.8"
services:
  web:
    image: nginx:1.19
    ports:
      - "80:80"
    deploy:
      replicas: 3
      update_config:
        parallelism: 1
        delay: 10s
      restart_policy:
        condition: on-failure

Deploying the Stack

Deploy the stack using this command:


docker stack deploy -c docker-compose.yml my_stack

Your service will be deployed in a swarm with 3 replicas, and you can update it with minimal downtime using rolling updates.
